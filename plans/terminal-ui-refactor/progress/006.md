## Steps done
- [x] Un-swallowed `childPids`/`psRows` exec errors: new module-level `isNoMatchExit(err)`
      guard checks `err.code === 1` (pgrep/ps "no match" convention) and returns `[]` only
      for that case; any other code or thrown non-exit error now re-throws as
      `TmuxExecError` with argv+stderr, matching the central `tmux()` helper's error shape.
- [x] Extracted the duplicated `TmuxExecError` try/catch into a shared module-level
      `execTmux(argv)` helper. The private `tmux()` method now delegates to it (prefixing
      `-S <socket>`); `newSession` builds its own argv with `-f /dev/null` BEFORE `-S`
      (the global-flag constraint, preserved verbatim, documented inline) and also routes
      through `execTmux` — no more copy-pasted catch block.
- [x] `term-wait-for.tool.ts`: added an inline comment documenting the INTENTIONAL
      non-reset of the read-cursor on match (never touches `ctx.cursor`) — contrasted with
      `term_snapshot`'s explicit tail-reset. No behavior change.
- [x] `term-ps.tool.ts:21`: `processes[]` now sourced from `ctx.tmux.descendantPids(panePid)`
      (full tree) instead of depth-1 `childPids`, matching `term_signal`'s tree boundary.
      `fgPid` is unchanged — still `childPids[0] ?? null` (direct foreground child).
- [x] Tests added:
      - `tmux.client.spec.ts`: +6 tests (childPids/psRows exit-1→[] vs non-1→TmuxExecError;
        newSession argv ordering (-f before -S) + DRY reuse fault-surfacing).
      - `term-ps.tool.spec.ts` (new file): grandchild pid appears in `processes[]` via
        `descendantPids` while `fgPid` stays the direct child; fgPid=null edge case.
