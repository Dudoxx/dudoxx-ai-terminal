## Deviations

- `EXEC_RUNNER` injection token added to SessionService to enable hermetic unit testing without `jest.spyOn` on promisified child_process (the CommonJS interception pitfall).
- `defaultExecRunner` implemented via `promisify(execFile)` wrapper; production behaviour unchanged.

## Recovery note (attempt 1 → fixed, 2026-06-28)

**Root cause**: B3/B4 validation used only `build && test`; `nest build` uses SWC (strips types) and `jest` uses ts-jest (lenient). Neither runs full `tsc --noEmit`. 7 type errors slipped through undetected.

**Fixes applied to `session.service.ts`**:
1. `defaultExecRunner` — was `promisify((file, args, cb) => execFile(file, args, cb))`. The hand-written 3-arg wrapper defeats Node's `execFile` `promisify.custom` overload (which resolves to `{stdout, stderr}`), causing TS to infer `Promise<string>` instead of `Promise<ExecResult>`. Fixed: `promisify(execFile)` directly.
2. `createTerminal()` — lines 183/191/192 passed `rawWindowId` (plain `string`) where branded `WindowId` was required by `resolvePids`/`resolveCwd`/`resolveCommand`. Fixed: pass the already-branded `windowId` const (line 167). `rawWindowId` remains only in the string interpolation at line 179 where a raw string is correct.
3. Added `resolveTerminalId(windowId: WindowId): TerminalId | undefined` reverse lookup — required to wire the control-mode `WindowIdResolver` correctly (see B4 recovery note).

**Re-validation gate**: `pnpm -F ddx-term-broker build && pnpm -F ddx-term-broker test && pnpm -F ddx-term-broker tsc:check` — all exit 0.
