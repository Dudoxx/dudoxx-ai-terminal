## Recovery note (attempt 2 → secfix-broker, 2026-06-28)

**Security review verdict**: FAIL (3 issues — HIGH-1 + MED-2 + MED-3). Full
diagnosis: `plans/ddx-terminal-bridge/handoffs/lead--to--secfix-broker.md`.

**Fix 1 — HIGH-1 — AC#12 cross-terminal isolation** (`term.gateway.ts`):
- Root cause: `handleClientMessage` dispatched input using `parsed.terminalId`
  from the MESSAGE BODY without verifying it against the socket's SUBSCRIBED
  terminalId. A client subscribed to A could inject into B.
- Fix: look up `this.connections.get(socket)` before dispatch; assert
  `sub.terminalId === parsed.terminalId`. On mismatch: `socket.close(1008,
  'terminalId mismatch')` + return. The `_socket` param of `handleInputFrame`
  renamed back to `_socket` (unused in body; isolation check runs in caller).
- Spec: 2 new tests in `TermGateway › AC#12 cross-terminal isolation` proving
  spoofed-terminalId is rejected (1008) and a legitimate frame is accepted.

**Fix 2 — MED-2 — WS CORS** (`term.gateway.ts`):
- Root cause: `@WebSocketGateway({ path: '/term' })` had no `cors` option;
  NestFactory global CORS does NOT apply to raw ws:// upgrades.
- Fix: add `cors: { origin: corsOrigins }` to the decorator. `corsOrigins`
  derives from `CORS_ORIGINS` env using the identical split/filter pattern
  as `main.ts` (reused, not duplicated).

**Fix 3 — MED-3 — title validation** (`terminal.controller.ts`):
- Root cause: `POST /api/v1/terminals` had no ValidationPipe; `title` was
  unbounded and flowed into `tmux new-window -n <title>` argv.
- Fix: inline `ZodPipe<T>` using `zod/v4`; `CreateTerminalBodySchema` bounds
  `title` to `z.string().max(64).optional()`. Removed re-export of
  `CreateTerminalDto` (unused after using the Zod-inferred type inline).

## Recovery note (attempt 1 → fixed, 2026-06-28)

**Root cause**: same as B3 — build+test-only gate missed full `tsc --noEmit`.

**Fixes applied to `terminal.service.ts`**:
- `defaultExec` had identical hand-written 3-arg wrapper bug as B3. Fixed: `promisify(execFile)` directly.

**Fixes applied to `term.gateway.ts`**:
- `onApplicationBootstrap()` wired `(windowId: WindowId) => this.sessionService.resolveWindowId(windowId)` as the `WindowIdResolver`. This is a real logic bug (FM#4): `resolveWindowId` takes a `TerminalId` and returns a `WindowId` — the opposite direction of what `WindowIdResolver` requires. The parser receives `%output @7 <data>` where `@7` is a `WindowId` and must resolve it to a `TerminalId` for routing. Fixed: wire `resolveTerminalId(windowId)` (reverse lookup added to SessionService in B3 fix). The brand types caught this conflation at compile time as intended.
- `afterInit(server)` assigned `this.server = server` redundantly (NestJS already injects into the `@WebSocketServer()` decorated field). Renamed param to `_server` and dropped the manual assignment. The decorated field is retained as the NestJS injection target.

**Re-validation gate**: `pnpm -F ddx-term-broker build && pnpm -F ddx-term-broker test && pnpm -F ddx-term-broker tsc:check` — all exit 0 (build: 22 files, tests: 42 passed across 4 suites).
