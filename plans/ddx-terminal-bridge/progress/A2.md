## Deviations

- **schema-lock.json shape** mirrors the sibling's intent (a lockable manifest of the
  exported contract) but its content is a per-schema signature+hash rather than an
  event-registry list — the sse-contract sibling is event-oriented; this package is
  schema-oriented, so the lock enumerates exported zod schemas. Generated from the built
  bundle so it always reflects reality. Not a new pattern (NEEDS_REVIEW not required) —
  same versioned-contract discipline, adapted to a schema-export package.
- **Added two frames beyond the shard's named five**: `process-snapshot` (server→client)
  is an additive helper for pushing pid snapshots to a renderer; the union is explicitly
  "open for v2" per the shard. The five required server→client variants
  (output/layout-change/window-add/window-close/error) + input are all present. Additive,
  non-breaking.
- **Added `WindowId` branded type** alongside `TerminalId` — the shard named TerminalId/
  PidRef; WindowId branding strengthens the identity/snapshot separation invariant at the
  type layer (windowId can't be passed where terminalId is required). Additive.
