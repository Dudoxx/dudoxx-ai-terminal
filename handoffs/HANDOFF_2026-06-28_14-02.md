### Goal B — New MCP capabilities + automatic wiring

The MCP exposes 9 verbs + 2 helpers today (term_create/list/destroy, send/read/wait-for/
signal/ps, panes/snapshot). Adding a NEW capability follows a fixed **4-layer wiring path**
(this is the reusable playbook — see §2). Candidate new verbs the user may want:
- **term_resize(terminalId, cols, rows)** — change pane size (pairs with Goal A resize UI).
- **term_split(terminalId, direction)** — tmux split-window; needs viewer multi-pane (Goal A).
- **term_write_file / term_read_file** — push/pull a file into the pane's cwd (convenience).
- **term_paste(terminalId, text)** — bracketed-paste a large blob (chunked, ordered).
- **term_rename(terminalId, title)** — rename a window/tab.
- **term_scrollback(terminalId, lines)** — explicit full-scrollback fetch (vs delta read).
- **term_env(terminalId)** — report the pane's env/cwd/git-branch for richer UI.

Each requires: contract schema → MCP tool → broker REST+WS → web viewer render. §2 is the
step-by-step.

## 2. THE ADD-A-CAPABILITY WIRING PLAYBOOK (reusable — for Goal B)

To add a new MCP verb end-to-end, touch these 4 layers IN ORDER (reciprocal-pair contract):

1. **Contract** (`packages/ddx-term-contract/src/`): add the tool I/O schema to
   `mcp-tools.ts` (zod/v4); if it needs a new WS frame, add it to `ws-frames.ts`. The MCP
   BUNDLES the contract at build (tsup inlines from source) — so contract changes need an MCP
   rebuild + republish if shipping. Keep `active`/optional-field discipline (the 0.1.3 fix:
   `.default(false)` for forgiving validation).
2. **MCP tool** (`ddx-term-mcp/src/tools/term-<verb>.tool.ts`): implement the verb; register
   it in `server.ts` tool list + `registry.ts`. The MCP is a THIN client — it resolves
   terminalId→windowId via the broker REST resolver (`registry-resolver.ts`) and either calls
   a broker REST endpoint OR shells tmux directly (NEVER hold a PTY — load-bearing invariant,
   `no-pty.spec.ts` enforces). Add a `.spec.ts`.
3. **Broker** (`ddx-term-broker/src/modules/terminal/`): add the REST endpoint to
   `terminal.controller.ts` + logic to `terminal.service.ts` (delegates registry to
   `session.service.ts`). If the verb produces a live frame for the browser, emit through
   `gateway/term.gateway.ts` (dispatchFrame → coalesce → flush). Add a `.spec.ts`.
4. **Web viewer** (`ddx-term-web/src/`): if the capability changes what the human sees, render
   it — handle the new WS frame in `xterm-client.ts handleServerFrame()` switch, or call the
   new REST endpoint via the proxy from `page.tsx` (browser → /api/v1/* → broker, never direct).

Verify: `pnpm --filter <pkg> tsc:check` + tests per package, then a BROWSER recheck (the hard
lesson — wire-green ≠ browser-working; see §Dead Ends).

## 3. KEY DECISIONS / INVARIANTS TO HONOR (don't relitigate)

- **MCP is PTY-free** — never add node-pty/pty.spawn to ddx-term-mcp (webmux trap, FM#1).
- **Broker owns canonical dims** — tmux pinned 120×30 (`default-size`); the WEB client now
  renders at exactly COLS×ROWS (xterm-client.ts COLS/ROWS consts) and MUST NOT fit-to-container
  (that desyncs cursor math — this session's D-6 fix). A `term_resize` verb must change BOTH
  the tmux session size AND the client consts in lockstep.
- **DOM renderer only** — WebGL removed (silent blank pane in embedded Chrome). Don't re-add.
- **Browser input = raw bytes via `send-keys -H`** (hex), serialized per-terminal. Don't revert
  to `send-keys -l` (breaks Ctrl-C/arrows/Enter).
- **Control-mode lines strip trailing CR** (PTY ONLCR fix) — keep it.
- **Node 22 for broker + web** (node-pty); MCP fine on any ≥20.
- **No auth in v1** (broker binds 127.0.0.1) — if ever exposed, auth guard FIRST.

## 4. DEAD ENDS (do NOT repeat)
- WebGL renderer — silent bg-only paint in embedded Chrome. DOM renderer is the answer.
- `send-keys -l` for interactive browser input — can't deliver control/escape bytes.
- Disabling zsh-syntax-highlighting/autosuggestions to fix live-echo garble — WRONG lead;
  real cause was a stray `\r` left on every control-mode `%output` line by the line-splitter
  (PTY ONLCR turns the protocol `\n` into `\r\n`). One-line fix in control-mode.attach.ts.
- `fitAddon.fit()` to container — desyncs cursor-relative redraws vs broker's pinned grid.
- Trusting the test suite for interactive AC — 5 of 8 fixes were for AC the wire passed but
  the browser failed. ALWAYS browser-verify interactive behavior.

## 5. BLOCKERS (fix before clean demos — both confirmed in 2× DEMO-TASKS runs)
- **0.C term_destroy not atomic**: tmux kill-window succeeds, broker DELETE returns 500 before
  evicting the registry entry → orphan (term_list still shows it; retry hits "can't find
  window"). Fix: make destroy idempotent (treat missing window as success, 200/204) and evict
  from registry FIRST or in a `finally`. File: `ddx-term-broker/src/modules/terminal/
  terminal.service.ts` destroy path + `session.service.ts`.
- **0.D registry↔tmux drift**: broker registry is in-memory; a broker restart loses all
  terminalId↔windowId mappings while tmux windows survive → `GET /terminals` 500s / stale
  tabs. Also accumulates orphans from 0.C. Fix: a boot-time reconcile (list tmux windows, drop
  registry entries whose window is gone, optionally adopt unknown windows). This is also why
  the web viewer can't reconnect to a terminal after a broker restart.
